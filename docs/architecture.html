System Architecture

Data Flow

1

JSON Data Files

Static data stored in server/data/*.json -- inventory, orders, demand forecasts, backlog, spending, transactions

2

mock_data.py loads at startup

All JSON files parsed and held in Python lists in memory. No database queries -- pure in-memory filtering.

3

FastAPI routes apply filters

apply_filters() handles warehouse, category, status. filter_by_month() supports month (2025-01) and quarter (Q1-2025) formats. Skips filter if value is "all".

4

Pydantic validates response

Every endpoint declares a response_model. Pydantic serializes and validates outgoing data before sending JSON.

5

api.js sends HTTP requests

Axios client at client/src/api.js maps filter state to query params. Omits params with value "all".

6

Composables manage shared state

useFilters() provides singleton reactive state. useAuth() holds mock user. useI18n() handles EN/JA translations.

7

Vue views render data

Views watch filter changes, re-fetch data, and render tables, KPI cards, SVG charts, and status badges. Detail modals shown on row click.

Filter System

FilterFrontend KeyAPI ParamOptions
Time Period selectedPeriod month all, 2025-01 through 2025-12, Q1-Q4
Warehouse selectedLocation warehouse all, San Francisco, London, Tokyo
Category selectedCategory category all, Circuit Boards, Sensors, Actuators, Controllers, Power Supplies
Order Status selectedStatus status all, Delivered, Shipped, Processing, Backordered

Filters are shared across all views via the useFilters() singleton composable. The FilterBar component binds to this state. API calls omit any parameter set to "all". Backend filtering is case-insensitive for categories.
